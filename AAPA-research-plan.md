# AAPA 研究方案与软件原型设计（先研究、后工程）

## 1. 研究问题固定（Problem Framing）

**目标定位**：AAPA 不是 COLONY 的通用替代，而是面向以下特定场景的全同胞家系归属算法：

- 已知候选父母本（候选家系集合可枚举）；
- 存在每个家系的 anchor 个体（已知高置信家系标签）；
- 存在大规模待测个体，需要高通量、可拒判的归属结果。

**核心研究问题**：

1. 在“已知父母本 + anchor + 大规模测试个体”条件下，AAPA 是否能在精度与可扩展性上优于或补充现有方法？
2. 在不同 SNP 数量、缺失率、基因分型错误率、anchor 数量、家系规模下，AAPA 的准确率、拒判质量和速度如何变化？
3. 当测试集中混入 unknown family 个体时，AAPA 的拒判机制是否能有效降低误判？

## 2. AAPA 方法框架

流程主线：

```text
Mendelian compatibility
+ anchor kinship
+ top-k pruning
+ confidence / rejection
```

### 2.1 输入与输出

**输入**：

- 基因型矩阵 `G`：`N × M`（N 个待测个体，M 个 SNP，推荐 dosage 编码 0/1/2，可含缺失）；
- 候选家系列表 `F`：每个家系含父本/母本 ID 及其基因型；
- anchor 表 `A`：anchor 个体 ID、家系标签、可选质量权重；
- 参数：`k`（top-k 候选数）、`tau_conf`（最小置信阈值）、`tau_rej`（拒判阈值）、缺失与错误容忍参数。

**输出**：

- 每个个体的 top-k 候选家系及分数；
- 最终分配标签（家系 ID 或 `REJECT`）；
- 置信度与拒判原因（如冲突过高、分数间隔不足、未知家系疑似）。

### 2.2 核心打分

设个体 `i`、家系 `f`、位点 `m`：

1. **Mendelian compatibility（冲突计数）**
   - 根据父母本基因型推导子代允许基因型集合；
   - 若 `G[i,m]` 不在允许集合内记冲突 `c_{i,f,m}=1`，否则 `0`；
   - 忽略缺失位点；
   - 得到归一化冲突率：
     \[
     C_{i,f}=\frac{\sum_m c_{i,f,m}}{\#\text{有效位点}}
     \]

2. **Anchor kinship（anchor 亲缘支持）**
   - 在家系 `f` 的 anchor 集合内计算 `i` 的亲缘相似性（IBS/kinship proxy）；
   - 聚合为 `K_{i,f}`（例如均值或稳健统计量）。

3. **综合分数（示例）**
   \[
   S_{i,f}= -\alpha C_{i,f} + \beta K_{i,f}
   \]
   其中 `\alpha, \beta` 可通过开发集或模拟数据调参。

### 2.3 Top-k 剪枝

- 先用 `C_{i,f}` 快速筛去明显不相容家系；
- 对剩余家系按 `S_{i,f}` 排序，保留 top-k；
- 在大 `F` 场景下显著减少后续计算与内存开销。

### 2.4 置信度与拒判规则

建议同时使用以下规则：

1. **绝对阈值拒判**：若最优家系 `f*` 的 `S_{i,f*} < tau_conf`，拒判；
2. **相对间隔拒判**：若 top1 与 top2 分差 `\Delta_i < tau_rej`，拒判；
3. **冲突上限拒判**：若 `C_{i,f*}` 超上限，拒判；
4. **unknown family 可疑拒判**：若所有候选家系均低置信或高冲突，标记 `UNKNOWN/REJECT`。

### 2.5 复杂度分析

- 朴素 Mendelian 冲突计算：`O(N × F × M)`；
- anchor 相似度（朴素）：`O(N × F × A_f × M)`，`A_f` 为家系 anchor 数；
- top-k 后可将精细计算限制在 `k << F` 的候选上。

工程优化重点：优先优化 `N × F × M` 冲突计算（后续 Rcpp 化）。

## 3. Benchmark 实验设计

## 3.1 对比方法

- **COLONY**：家系/亲缘重建基线；
- **KING / PLINK 最近亲缘 baseline**：基于最近亲缘或距离规则进行家系归属。

## 3.2 因素与场景矩阵

至少覆盖：

- SNP 数：如 `M ∈ {500, 2k, 10k, 50k}`；
- anchor 数：每家系 `A_f ∈ {1, 2, 5, 10}`；
- 缺失率：`{0%, 1%, 5%, 10%}`；
- 错误率：`{0%, 0.1%, 0.5%, 1%}`；
- 家系数量：`F` 从小到大（如 10, 50, 200）；
- 测试个体规模：`N` 从千到十万级分层；
- 测试集混入 unknown family 个体（如 5%、10%、20%）。

## 3.3 评估指标

- 分类准确率（仅已知家系样本）；
- 宏/微平均 F1；
- 拒判率、拒判精确性（被拒个体中 truly unknown 或低置信比例）；
- 误判率（尤其 unknown 被误分配到已知家系）；
- 运行时间、峰值内存、吞吐量（样本/秒）；
- 校准质量（置信分数与真实正确率的一致性）。

## 3.4 实验输出

- 主结果表：不同方法 × 场景；
- 灵敏度分析图：准确率/误判率随 SNP、anchor、缺失率、错误率变化；
- 可扩展性曲线：时间与内存随 `N/F/M` 增长关系。

## 4. R 包结构（建议）

```text
aapa/
  R/
    assign.R
    qc.R
    scoring.R
    plot.R
  src/
    mendelian.cpp
    anchor_kinship.cpp
    topk.cpp
  inst/extdata/
    small_example/
  tests/testthat/
  vignettes/
```

模块职责建议：

- `R/assign.R`：主流程接口、参数校验、结果对象；
- `R/qc.R`：缺失率过滤、位点/样本质量控制；
- `R/scoring.R`：冲突、亲缘、综合分数与拒判规则；
- `R/plot.R`：分数分布、top-k、拒判诊断可视化；
- `src/*.cpp`：性能关键路径（先占位，后迁移）。

## 5. MVP（最小可用版本）实施范围

第一阶段仅实现：

1. 读取 genotype dosage 矩阵；
2. 读取 parents 表；
3. 读取 anchors 表；
4. 计算 Mendelian conflict；
5. 输出 top-k 家系；
6. 加入拒判逻辑。

**MVP 非目标**：不先做复杂联合推断、不先追求全量 C++ 优化、不先覆盖所有可视化。

---

## 6. Rcpp 加速路线（第二阶段）

策略：**先 R 版做对，再迁移最慢环节**。

1. 在 R 中完成端到端正确实现与单元测试；
2. profiling 定位瓶颈（预期为 `N × F × M` 冲突循环）；
3. 将冲突计算迁移到 `src/mendelian.cpp`（必要时向量化/并行）；
4. 保持 R 接口与结果一致，回归测试确保数值等价；
5. 逐步迁移 anchor 亲缘与 top-k 关键路径。
